# 도서관 관리 시스템 (LMS) — 요구사항 정의 및 기능명세서

**프로젝트명:** KT AIVLE School Mini Project 4 — 도서관 관리 시스템  
**작성일:** 2026-05-26  
**기술 스택:** React 19 · Vite · json-server · React Router DOM v7 · Framer Motion · OpenAI gpt-image-1

---

## 1. 프로젝트 개요

3D 복도 형태의 도서관 UI를 통해 소장 도서를 탐색하고, 관심 도서를 책장에 담으며, CRUD 기능과 AI 표지 이미지 자동 생성을 제공하는 웹 기반 도서관 관리 시스템이다.

---

## 2. 사용자 구분

| 구분 | 역할 |
|------|------|
| 일반 이용자 | 3D 복도 탐색, 도서 목록 조회/검색/필터, 도서 상세 확인, 내 책장(좋아요) 관리 |
| 관리자 | 도서 등록·수정·삭제, AI 표지 이미지 생성 |

## 4. 기능명세서

### 4.1 3D 도서관 복도 (FR-01, FR-02)

**화면:** `/` (LibraryScene → CorridorView ↔ ShelfView)

| 항목 | 내용 |
|------|------|
| 복도 뷰 | 장르별 책장이 3D 원근감으로 나열된 복도 화면 |
| 책장 선택 | 장르 책장 클릭 → ShelfView로 전환 (Framer Motion 애니메이션) |
| 책장 뷰 | 선택 장르 도서를 책등 색상과 함께 3D 책장 형태로 표시 |
| 뒤로 가기 | ShelfView → CorridorView 복귀 버튼 |
| 책 선택 | ShelfView에서 책 클릭 → `/books/:id` 상세 페이지 이동 |
| 데이터 | `GET /books?genreCode={code}` 로 장르 도서 조회 |

---

### 4.2 도서 목록 조회 (FR-03 ~ FR-06)

**화면:** `/books` (BookList)

| 항목 | 내용 |
|------|------|
| 진입 시 동작 | `GET /books?_sort=createdAt&_order=desc` (최신순) |
| 카드 표시 항목 | 표지 이미지(2:3, 없으면 장르 컬러 플레이스홀더), 장르 태그, 제목, 저자, 등록일 / 좋아요 여부(❤ 뱃지) |
| 검색 | 제목 부분 일치, 입력 후 250ms 디바운스 → `title_like={q}` 쿼리 |
| 장르 필터 | 상위 장르 선택 → 하위 장르 셀렉트 활성화; `genreCode_like={topCode}` 또는 `genreCode={subCode}` |
| 좋아요 필터 | "❤️ 책장만" 토글 버튼 → `isLiked=true` 쿼리 추가 |
| 필터 조합 | 검색·장르·좋아요 조건 AND 조합, 서버에 단일 요청으로 처리 |
| 빈 결과 | "조건에 맞는 책이 없어요." 안내 메시지 |
| 연결 실패 | "json-server에 연결할 수 없어요." + `npm run server` 안내 |

---

### 4.3 도서 상세 조회 (FR-07, FR-08, FR-09)

**화면:** `/books/:id` (BookDetail)

| 항목 | 내용 |
|------|------|
| 진입 시 동작 | `GET /books/:id` 단건 조회 |
| 표시 항목 | 표지 이미지(없으면 장르 컬러 플레이스홀더), 장르 태그, 제목, 저자, 등록일·수정일, 본문 내용 |
| 좋아요 버튼 | "🤍 책장 담기" / "❤️ 책장에 담김" 토글; `PATCH /books/:id { isLiked, updatedAt }` |
| 수정 버튼 | `Link` → `/books/:id/edit` |
| 삭제 버튼 | `window.confirm("'제목'을(를) 삭제할까요?")` 확인 후 `DELETE /books/:id` → `/books` 이동 |
| 뒤로 가기 | `Link` → `/books` |

---

### 4.4 AI 표지 이미지 생성 (FR-14, FR-15, FR-16)

**위치:** BookDetail 페이지 내 AI 패널 (토글 방식)

| 항목 | 내용 |
|------|------|
| 패널 열기 | "✨ AI 표지 생성" 버튼 클릭 시 패널 토글 |
| API Key 입력 | 사용자가 직접 OpenAI API Key 입력 (password 타입); `localStorage`에 저장하여 재입력 생략 |
| 품질 선택 | low(빠름) / medium(권장) / high(고품질) |
| 생성 | 책 제목·저자·장르·본문을 바탕으로 프롬프트 조합 → OpenAI `gpt-image-1` 호출 (1024×1536, portrait) |
| 결과 | base64 Data URL을 미리보기 이미지로 즉시 표시 |
| 저장 | "💾 이 표지로 저장" → `PATCH /books/:id { coverImageUrl, updatedAt }` |
| 재생성 | "🔄 다시 생성" 버튼으로 동일 설정으로 재호출 |
| 오류 처리 | API Key 미입력 / 연결 실패 / 응답 오류 시 패널 내 오류 메시지 표시 |

---

### 4.5 도서 등록 (FR-11)

**화면:** `/books/new` (BookCreate)

| 항목 | 내용 |
|------|------|
| 입력 필드 | 제목(필수), 작가명(필수), 장르 — 상위·하위 2단계(필수), 본문 내용(필수) |
| 유효성 검사 | 필수 항목 미입력 시 "제목 · 작가 · 장르 · 본문은 모두 필수예요." 오류 표시 |
| 글자 수 표시 | 본문 입력란 하단에 현재 글자 수 실시간 표시 |
| 등록 API | `POST /books` — `coverImageUrl: ''`, `isLiked: false`, `createdAt/updatedAt` 자동 추가 |
| 등록 완료 | 생성된 도서 상세 페이지(`/books/:id`)로 이동 |
| 취소 | `/books`로 이동 |

---

### 4.6 도서 수정 (FR-12)

**화면:** `/books/:id/edit` (BookEdit)

| 항목 | 내용 |
|------|------|
| 진입 시 동작 | 기존 도서 데이터로 폼 pre-fill (상위·하위 장르 2단계 복원) |
| 수정 API | `PATCH /books/:id` — 변경 필드 + `updatedAt` 갱신 |
| 수정 완료 | 도서 상세 페이지(`/books/:id`)로 이동 |
| 취소 | `/books`로 이동 |

---

### 4.7 도서 삭제 (FR-13)

**위치:** BookDetail 페이지 내 "🗑️ 삭제" 버튼

| 항목 | 내용 |
|------|------|
| 확인 절차 | `window.confirm("'제목'을(를) 삭제할까요?")` 팝업 |
| 삭제 API | `DELETE /books/:id` |
| 삭제 완료 | `/books` (도서 목록)으로 이동 |
| 취소 | confirm 취소 시 아무 동작 없음 |

## 6. 데이터 모델

### Book
| 필드 | 타입 | 설명 |
|------|------|------|
| id | number | 자동 부여 (json-server) |
| title | string | 도서 제목 |
| author | string | 저자명 |
| genreCode | string | 하위 장르 코드 (예: `NV-01`) |
| content | string | 도서 소개 / 본문 |
| coverImageUrl | string | 표지 이미지 URL 또는 base64 Data URL (없으면 빈 문자열) |
| isLiked | boolean | 좋아요 여부 |
| createdAt | string | 등록 일시 (ISO 8601) |
| updatedAt | string | 수정 일시 (ISO 8601) |

### Genre
| 필드 | 타입 | 설명 |
|------|------|------|
| code | string | 장르 코드 (예: `NV`, `NV-01`) |
| label | string | 장르명 |
| parentCode | string \| undefined | 상위 장르 코드 (없으면 최상위) |

## 7. API 명세 (json-server)

| Method | Endpoint | 주요 쿼리 파라미터 | 설명 |
|--------|----------|-------------------|------|
| GET | `/books` | `title_like`, `genreCode`, `genreCode_like`, `isLiked`, `_sort`, `_order` | 도서 목록 (필터·정렬 지원) |
| GET | `/books/:id` | — | 도서 단건 조회 |
| POST | `/books` | — | 도서 등록 |
| PATCH | `/books/:id` | — | 도서 부분 수정 (좋아요·표지·수정 공용) |
| DELETE | `/books/:id` | — | 도서 삭제 |
| GET | `/genres` | — | 전체 장르 목록 |
